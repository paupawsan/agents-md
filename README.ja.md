# agents-md

AIコーディングエージェント向けのシステムとツールのコレクション。実務経験に基づいて構築されています。現在は堅牢なメモリシステムとクリティカルシンキングフレームワークを含み、将来的にはさらに多くのシステムが追加される予定です。

## 🚀 クイックスタート

**初心者ですか？** → **[シンプルセットアップガイド](docs/simple-setup.ja.md)** - 技術者以外の方のためのステップバイステップガイド

**経験豊富な開発者ですか？** → **[セットアップガイド](docs/setup-guide.ja.md)** を参照

### 必要なもの

1. **ダウンロード** リポジトリ（ZIPダウンロードまたは`git clone`）
2. **選択** メモリディレクトリ名（例：`my-memory`）
3. **設定** 選択した名前で`AGENTS.md`を設定
4. **追加** メモリフォルダをCursor/VS Codeワークスペースに追加
5. **初期化** メモリを初期化: `"initialize memory for this project"`

**以上です！** AIエージェントがプロジェクトのパターン、設定、決定を自動的に記憶するようになります。

**⚠️ 重要**: このリポジトリ全体が**設定テンプレート**です。エージェントはagents-mdプロジェクト構造のいかなる部分（メモリシステム、ドキュメント、`_agents-md/`ディレクトリを含む）も自動的に作成、コピー、または再構築しません。まずシステムを設定し（例：`AGENTS.md`の`MEMORY_PATH`）、その後明示的に初期化を要求する必要があります。

## 🧠 メモリシステム

マルチプラットフォーム対応の堅牢なRAG最適化メモリ管理システム。

### 主な機能
- **高度なRAGサポート**: 80-95%のトークン削減を実現するインテリジェントなセマンティック検索
- **マルチプラットフォーム**: macOS、Linux、Windowsのネイティブサポート
- **プライバシー重視**: 人間が読めるMarkdownファイルによるローカルストレージ
- **エージェントAI最適化**: セマンティック検索を実行できる高度なAIモデル向けに設計

### ワークフローオプション
- **構造化ワークフロー**: トピック、セッション、インデックスを含む完全なメモリシステム（複雑なプロジェクトに最適）
- **シンプルワークフロー**: クイックな日次ログ用の単一ファイルテンプレート（ビベコーディングに最適）

### なぜローカルMarkdownファイルなのか？

- ✅ **完全な透明性**: 任意のテキストエディタで編集できる人間が読めるファイル
- ✅ **依存関係なし**: データベース、API、外部サービスは不要
- ✅ **バージョン管理に適している**: Gitと完璧に連携
- ✅ **完全な制御**: メモリはあなたのマシンに保存されます
- ✅ **簡単なバックアップ**: ディレクトリをコピーするだけ

### モデル互換性

⚠️ **エージェント対応モデルが必要**（Claude Sonnet/Opus、GPT-4など）：
- 複雑な指示を理解できる
- セマンティック検索と取得を実行できる
- メモリ読み込みについてインテリジェントな決定を行える
- 自律的にマルチステップワークフローを実行できる

## 🧠 クリティカルシンキングシステム

証拠に基づく推論、事実検証、インテリジェントな意思決定のための包括的なフレームワーク。

### 主な機能
- **コンテキスト認識厳格さ**: リスクとコンテキストに基づいて検証レベルを調整（セキュリティ重要 vs プロトタイピング）
- **証拠に基づく推論**: 仮定ではなく検証で裏付けられた決定
- **幻覚防止**: 誤情報を避けるための構造化検証プロトコル
- **敬意ある異議申し立て**: 適切な場合に仮定を建設的に挑戦

### コア機能
- **事実検証**: グラウンドチェックと検証ワークフロー
- **マルチ視点分析**: 技術的、人的、ビジネス要因を考慮
- **リスク評価**: 高リスク vs 低リスクシナリオに適した厳格さ
- **コミュニケーション適応**: 専門家 vs 非技術者向けにスタイルを調整

### 活性化タイミング
- **高リスクシナリオ**: 銀行アプリ、ヘルスケアシステム、本番データベース変更 → 最大限の検証
- **低リスクシナリオ**: プロトタイピング、デバッグ、ドキュメント → 柔軟で協調的
- **不確実な状況**: 事実の検証が必要な場合や仮定に挑戦すべき場合
