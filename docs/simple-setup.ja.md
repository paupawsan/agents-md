# シンプルセットアップガイド - 技術者以外の方へ

**プログラマーでなくても心配いりません！** このガイドでは、専門用語を避け、平易な言葉でメモリシステムのセットアップをステップバイステップで説明します。

## これは何ですか？

これはAIコーディングアシスタント（Cursorなど）用の**メモリシステム**です。AIアシスタントがプロジェクトについて覚えておけるようにし、時間の経過とともにあなたとより良く協力できるようにします。

**例えるなら**: AIアシスタントがプロジェクトについて学んだことを書き留めるノートブックのようなものです。忘れないようにするためです。

## 必要なもの

- コンピューター（Mac、Windows、またはLinux）
- CursorまたはVS Codeエディタがインストールされていること
- テキストエディタの基本的な使い方（ファイルを開いたり保存したりする）

**必要ないもの**:
- プログラミングの知識
- コマンドライン/ターミナルの経験（ただし、簡単なコマンドは説明します）
- Pythonやその他のプログラミング言語

---

## ステップバイステップセットアップ

### ステップ1: ファイルをダウンロードまたは取得

**オプションA: ZIPとしてダウンロード（最も簡単 - 技術者以外の方に推奨）**

GitHubにいる場合：
1. リポジトリページの上部にある緑色の**「Code」**ボタンをクリック
2. **「Download ZIP」**をクリック
3. ZIPファイルがコンピューターにダウンロードされます（通常はダウンロードフォルダ）
4. ダウンロードしたZIPファイルを見つけて、ダブルクリックして展開/解凍します
5. `agents-md`（または`agents-md-main`）という名前のフォルダが表示されます - これに必要なすべてのファイルが含まれています

**Macの場合**: ZIPファイルをダブルクリックすると自動的に展開されます
**Windowsの場合**: ZIPファイルを右クリック → 「すべて展開」 → 場所を選択
**Linuxの場合**: ZIPファイルを右クリック → 「ここに展開」またはアーカイブマネージャーを使用

**オプションB: すでにファイルを持っている場合**
- すでにファイルを持っています！`agents-md`フォルダを保存した場所を見つけてください。

**オプションC: Gitを使用している場合（開発者向け）**
- このステップをスキップしてください - すでにファイルを持っています。

**見えるはずのもの**: `AGENTS.md`、`README.md`、`_agents-md`などのファイルがあるフォルダ

---

### ステップ2: メモリフォルダの名前を選択

AIのメモリが保存される場所のシンプルな名前を考えてください。例：
- `my-ai-memory`
- `cursor-memory`
- `ai-notes`

**名前のルール**:
- 小文字、数字、ハイフン（-）、アンダースコア（_）のみ使用
- スペースなし
- 短くシンプルに

**良い名前の例**: `my-memory`、`ai-notes`、`cursor_memory`
**悪い名前の例**: `My Memory`（スペースと大文字あり）、`memory!`（特殊文字あり）

---

### ステップ3: AGENTS.mdファイルを設定

これはAIアシスタントにメモリの使い方を伝えるメインファイルです。

#### 3a. AGENTS.mdを開く

1. `agents-md`フォルダを開く
2. `AGENTS.md.en`（英語用）または`AGENTS.md.ja`（日本語用）というファイルを見つける
3. 右クリックして「開くアプリケーション」→ テキストエディタ（Cursor、VS Code、またはNotepad/TextEdit）を選択

#### 3b. 言語を選択

**英語の場合**:
- `AGENTS.md.en`をテンプレートとして使用します

**日本語の場合**:
- `AGENTS.md.ja`をテンプレートとして使用します

#### 3c. テンプレートをコピー

1. テンプレートファイル（`AGENTS.md.en`または`AGENTS.md.ja`）を開く
2. すべてのテキストを選択（Macでは`Cmd+A`、Windows/Linuxでは`Ctrl+A`）
3. コピー（`Cmd+C`または`Ctrl+C`）
4. 同じフォルダに`AGENTS.md`という新しいファイルを作成
5. 内容を貼り付け（`Cmd+V`または`Ctrl+V`）
6. ファイルを保存

**今やったこと**: テンプレートから独自の`AGENTS.md`ファイルを作成しました。

#### 3d. プレースホルダーを置き換える

次に、`{MEMORY_DIR}`をステップ2で選択した名前に置き換える必要があります。

1. `AGENTS.md`ファイルで、`Cmd+F`（Mac）または`Ctrl+F`（Windows/Linux）を押して検索を開く
2. 検索ボックスに`{MEMORY_DIR}`と入力
3. 「置換」または「すべて置換」をクリック
4. 選択した名前（例：`my-memory`）を入力
5. 「すべて置換」をクリック
6. ファイルを保存

**例**: `my-memory`を選択した場合、すべての`{MEMORY_DIR}`が`my-memory`になります

---

### ステップ4: プロジェクトでファイルを設定

メモリシステムを設定するには2つの方法があります。最適な方法を選択してください：

#### オプションA: ワークスペースセットアップ（簡単 - Cursor/VS Code推奨）

**CursorまたはVS Codeを使用している場合**、ファイルをコピーする代わりに`agents-md`フォルダをワークスペースに追加できます：

1. **CursorまたはVS Codeを開く**
2. **プロジェクトフォルダを開く**（ファイル → フォルダを開く）
3. **agents-mdをワークスペースに追加**:
   - `Cmd+Shift+P`（Mac）または`Ctrl+Shift+P`（Windows/Linux）を押す
   - 入力: `Add Folder to Workspace`
   - 「Add Folder to Workspace」を選択
   - `agents-md`フォルダに移動して選択
   - 「追加」または「フォルダを選択」をクリック

4. **AGENTS.mdが自動的に検出されます**:
   - 最近のバージョンのCursorは、ワークスペース内の`AGENTS.md`ファイルを自動的に見つけます
   - `agents-md`フォルダ内の`AGENTS.md`ファイルが自動的に使用されます
   - `AGENTS.md`をプロジェクトにコピーする必要はありません！

5. **`_agents-md`フォルダをコピー**（まだ必要）:
   - `_agents-md`フォルダをプロジェクトフォルダにコピーする必要があります
   - または、ワークスペースにも追加できます（今追加した`agents-md`フォルダに既に含まれています）

**これが簡単な理由**:
- ✅ 各プロジェクトに`AGENTS.md`をコピーする必要がない
- ✅ `agents-md`フォルダを更新すると自動的に更新される
- ✅ 複数のプロジェクトで簡単に動作する

#### オプションB: ファイルをコピー（従来の方法）

次に、メモリシステムを使用するプロジェクトフォルダに2つのものをコピーする必要があります。

#### コピーするもの：

1. **`_agents-md`フォルダ** - AIへの指示が含まれています
2. **`AGENTS.md`ファイル** - 今作成したファイルです

#### コピー方法：

**Macの場合**:
1. Finderを開く
2. `agents-md`フォルダに移動
3. `_agents-md`フォルダをコピー（右クリック → コピー）
4. `AGENTS.md`ファイルをコピー（右クリック → コピー）
5. プロジェクトフォルダに移動
6. 両方を貼り付け（右クリック → 貼り付け）

**Windowsの場合**:
1. エクスプローラーを開く
2. `agents-md`フォルダに移動
3. `_agents-md`フォルダをコピー（右クリック → コピー）
4. `AGENTS.md`ファイルをコピー（右クリック → コピー）
5. プロジェクトフォルダに移動
6. 両方を貼り付け（右クリック → 貼り付け）

**Linuxの場合**:
1. ファイルマネージャーを開く
2. `agents-md`フォルダに移動
3. `_agents-md`フォルダと`AGENTS.md`ファイルをコピー
4. プロジェクトフォルダに移動
5. 両方を貼り付け

**重要**: プロジェクトにすでに`AGENTS.md`ファイルがある場合、置き換えないでください！代わりに、両方のファイルを開き、新しい`AGENTS.md`の内容を既存のファイルの末尾にコピーしてください。

---

### ステップ5: メモリの保存場所を設定

AIのメモリには住む場所が必要です。これは通常、プロジェクトフォルダの外にあります。

#### 場所を選択：

**推奨場所**:
- **Mac**: `~/Documents/my-memory`（または選択した名前）
- **Windows**: `C:\Users\YourName\Documents\my-memory`
- **Linux**: `~/Documents/my-memory`

**またはクラウドストレージを使用**:
- **Google Drive（Mac）**: `~/Library/CloudStorage/GoogleDrive-your-email/My Drive/AI/my-memory`
- **iCloud（Mac）**: `~/Library/Mobile Documents/com~apple~CloudDocs/AI/my-memory`

#### メモリフォルダを作成：

1. ファイルマネージャーを開く（MacではFinder、Windowsではエクスプローラー）
2. 選択した場所（例：Documentsフォルダ）に移動
3. 選択した名前（例：`my-memory`）で新しいフォルダを作成
4. このフォルダへの完全なパスを覚えておく - 次のステップで必要になります

#### パスでAGENTS.mdを更新：

1. プロジェクトフォルダで`AGENTS.md`を開く
2. 上部の**設定**セクションを見つける
3. `**MEMORY_PATH**: `/path/to/your/memory-root`` という行を見つける
4. `/path/to/your/memory-root`を実際のメモリフォルダパスに置き換える

**例**: MacのDocumentsに`my-memory`を作成した場合、次のように置き換えます：
```
**MEMORY_PATH**: `~/Documents/my-memory`
```

**注意**: この1箇所だけパスを置き換えれば十分です。AIエージェントはそれがすべてに適用されることを理解します。

**心配しないでください** - デフォルトのパスは通常機能します！メモリフォルダを特別な場所に配置した場合のみ変更してください。

---

### ステップ6: メモリフォルダをCursor/VS Codeワークスペースに追加

**これが重要な理由**: CursorとVS Codeはワークスペース内のファイルにのみアクセスできます。メモリフォルダがプロジェクトの外にあるため、ワークスペースに追加する必要があります。

#### 方法：

1. CursorまたはVS Codeを開く
2. プロジェクトフォルダを開く（ファイル → フォルダを開く）
3. `Cmd+Shift+P`（Mac）または`Ctrl+Shift+P`（Windows/Linux）を押してコマンドパレットを開く
4. 入力: `Add Folder to Workspace`
5. リストから「Add Folder to Workspace」を選択
6. メモリフォルダ（ステップ5で作成したもの）に移動して選択
7. 「追加」または「フォルダを選択」をクリック

**見えるはずのもの**: メモリフォルダがCursor/VS Codeのサイドバーにプロジェクトフォルダと並んで表示されるはずです。

**トラブルシューティング**: フォルダが見つからない場合：
- まず作成したことを確認してください（ステップ5）
- ファイルブラウザーに完全なパスを入力してみてください
- Macでは、`Cmd+Shift+.`を押して隠しフォルダを表示する必要があるかもしれません

---

### ステップ7: テストしてみましょう！

すべてが機能することを確認しましょう：

1. プロジェクトでCursorまたはVS Codeを開く
2. メモリフォルダがサイドバーに表示されていることを確認
3. チャット/コマンド領域で、入力: `このプロジェクトのメモリを初期化`
4. AIがメモリフォルダにいくつかのファイルを作成するはずです

**成功の見た目**:
- メモリフォルダに新しいフォルダとファイルが表示される
- AIがメモリを設定していると応答する
- エラーメッセージが表示されない

**何か問題が発生した場合**:
- `AGENTS.md`のメモリフォルダパスが正しいことを確認
- メモリフォルダがワークスペースに追加されていることを確認（ステップ6）
- AIに尋ねてみてください：「[あなたのパス]のメモリフォルダにアクセスできますか？」

メモリシステムがセットアップされました！AIアシスタントは自動的に：
- プロジェクトについて覚えておく
- あなたの設定から学ぶ
- 重要なパターンとソリューションを保存する

## よくある質問

### Q: 他に何かする必要がありますか？

**A**: いいえ！一度セットアップすれば、メモリシステムは自動的に動作します。AIは必要に応じてメモリを保存および取得します。

### Q: 後でメモリフォルダの場所を変更できますか？

**A**: はい！`AGENTS.md`のパスを更新し、新しいフォルダがワークスペースに追加されていることを確認してください。

### Q: 間違えた場合はどうすればよいですか？

**A**: 心配しないでください！いつでも：
- ファイルの新しいコピーで最初からやり直す
- AIに問題の修正を依頼する
- より詳細な情報についてはメインのREADME.mdを確認する

### Q: すべての技術的な詳細を理解する必要がありますか？

**A**: いいえ！このシンプルなセットアップで十分です。README.mdの技術的な詳細はオプションで、上級ユーザー向けです。

### Q: 複数のプロジェクトでこれを使用できますか？

**A**: はい！各プロジェクトは独自の`AGENTS.md`を持つことができますが、すべて同じメモリフォルダを共有できます。AIは自動的にプロジェクトごとにメモリを整理します。

行き詰まった場合：
1. より詳細な手順についてはメインの`README.md`ファイルを確認してください
2. AIアシスタントにヘルプを依頼してください（ガイドできるはずです）
3. すべてのステップが正しく完了したことを確認してください

## 次は何ですか？

セットアップ後、次のことができます：
- AIアシスタントの使用を開始 - 自動的にメモリを使用します
- AIにプロジェクトについて特定のことを覚えておくよう依頼する
- メモリフォルダを閲覧してAIが学んだことを確認する
- `AGENTS.md`で設定をカスタマイズする（上級者向け）

**覚えておいてください**: メモリシステムは自動的に動作するように設計されています。特別なことは何もする必要はありません - 通常どおりAIアシスタントを使用するだけです！

## ⚠️ 重要: エージェント権限

**メモリシステムが動作する前に**、AIアシスタントにファイルコマンドを実行する権限を与える必要があります。

### 必要なコマンドは何ですか？

AIアシスタントは、メモリファイルを作成および管理するために、安全なコマンドを実行する必要があります：
- `mkdir` - メモリ用のフォルダ作成
- `touch` - ファイル作成
- `cat`, `head`, `grep` - メモリファイルの読み取り
- `echo`, `printf` - メモリファイルへの書き込み
- `cp`, `mv`, `ls` - メモリファイルの整理

### 権限を付与する方法

**Cursorの場合**:
1. 設定（歯車アイコン）に移動
2. 「Agent」または「AI」設定を探す
3. 「Command Execution」または類似の項目を探す
4. 上記のコマンドを許可コマンドに追加
5. または、AIが権限を求めたら「Allow」をクリック

**VS Codeの場合**:
- AIアシスタント拡張機能の設定を確認
- コマンド実行権限を探す

**アンチウイルスソフトウェアの場合**:
- 一部のアンチウイルスプログラムがこれらの操作をブロックする可能性があります
- プロジェクトフォルダを例外として追加
- エディタからのファイル操作を許可

### これが必要な理由は？

- **セキュリティ**: エディタは安全を守るためにコマンド実行を防ぎます
- **ファイルアクセス**: メモリファイルはプロジェクト外に保存されます
- **安全な操作**: これらのコマンドはファイルでのみ動作 - ソフトウェアのインストールやインターネットアクセスはできません

**心配しないでください**: これらはファイル操作だけです。AIは危険なコマンドを実行できません - 安全なファイルの読み取り/書き込みだけです。
